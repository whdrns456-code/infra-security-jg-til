1. 파일 내용 확인 및 조회
cat: 파일의 전체 내용을 터미널에 출력함. 텍스트 파일용이며 바이너리 파일은 깨져서 보임.

file: 파일이 텍스트인지, 실행 파일인지, 데이터 파일인지 형식을 확인해줌. (예: file /etc/hosts)

more: 페이지 단위로 읽기. (Space: 다음 페이지, Enter: 한 줄씩, b: 이전 페이지)

less: more와 비슷하지만 기능이 더 많음. (방향키 사용 가능, 파일 끝에 도달해도 종료되지 않음)

head: 파일의 앞부분 출력. 기본 10줄. (-n [줄수] 옵션으로 조절)

tail: 파일의 뒷부분 출력. 로그 확인 시 필수. (-f 옵션을 쓰면 실시간 업데이트 확인 가능)

wc: 파일의 통계 확인.

-l: 줄 수 (Line)

-w: 단어 수 (Word)

-m: 문자 수 (Character)

-c: 바이트 수 (Byte)

2. 디렉터리 및 파일 관리
touch:

파일이 없으면 0바이트의 빈 파일 생성.

파일이 있으면 타임스탬프(Access, Modification 시간)를 현재 시간으로 업데이트.

-t [[CC]YY]MMDDhhmm[.ss]: 특정 시간으로 강제 수정.

mkdir: 디렉터리 생성.

-p: 상위 디렉터리가 없으면 중간 경로까지 한꺼번에 생성.

stat: 파일의 i-node 번호, 권한, 접근/수정/변경 시간을 상세히 출력하는 고급 조회 명령어.

3. 복사 및 이동 (cp, mv)
cp (copy): 파일이나 디렉터리 복사.

cp [원본] [대상]: 파일 복사.

cp -r: 디렉터리 복사 시 필수 (Recursive). 하위 내용물까지 모두 복사함.

alias 설정 때문에 보통 -i 옵션이 적용되어 있어 덮어쓰기 전 물어봄 (y로 응답).

mv (move): 파일 이동 또는 이름 변경.

같은 디렉터리 내에서 이동하면 이름 변경 효과.

원본이 삭제되고 대상 위치로 옮겨짐.

주의: 디렉터리로 파일을 덮어쓰는 것은 불가능함.

4. 삭제 (rm)
rm (remove): 파일이나 디렉터리 삭제.

-d: 비어있는 디렉터리 삭제.

-r: 디렉터리 및 그 내부 내용물을 모두 삭제.

-f: 강제 삭제 (질문하지 않음).

-rf: 디렉터리 강제 삭제. 가장 강력하고 위험한 명령어이므로 경로 확인 필수.

5. 링크 (Link)
i-node: 리눅스 파일 시스템에서 파일을 식별하는 고유 번호 (ls -i로 확인).

하드 링크 (Hard Link): ln [원본] [링크]

원본과 동일한 i-node 번호를 가짐.

원본 파일을 지워도 하드 링크 파일이 남아있다면 데이터는 삭제되지 않음.

같은 파일 시스템 내에서만 생성 가능.

심볼릭 링크 (Symbolic Link): ln -s [원본] [링크]

윈도우의 '바로가기'와 같음. 새로운 i-node 생성.

원본의 경로를 가리키고 있으므로, 원본이 삭제되면 링크가 깨짐.

디렉터리나 다른 파일 시스템에 대해서도 생성 가능.
**핵심 개념 2: 오후 수업 리눅스 기초 명령어**
- 파일 링크 (Link)

- 하드 링크 (Hard Link) 
    - 명령어: ln [원본] [대상] 
    - 원본 파일과 동일한 i-node 번호를 공유함. 
    - 데이터 블록을 공유하므로 디스크 공간을 추가로 차지하지 않음. 
    - 파일 시스템이 같아야 생성 가능하며, 디렉터리는 링크 불가. 
    - 원본 파일이 삭제되어도 하드 링크 파일은 그대로 유지됨.

- 심볼릭 링크 (Symbolic Link) 
    - 명령어: ln -s [원본] [대상] 
    - 원본을 가리키는 새로운 i-node를 가진 별도의 파일 생성. 
    - 윈도우의 바로가기 아이콘과 유사한 개념. 
    - 다른 파일 시스템 간에도 생성 가능하며 디렉터리도 가능함. 
    - 원본이 삭제되면 링크가 끊어져 데이터에 접근할 수 없음(Broken Link).

- 와일드카드 (Wildcard)

    - 특수 문자 활용 
        - * : 0개 이상의 모든 문자 매칭. 
        - ? : 정확히 1개의 문자 매칭. 
        - [] : 대괄호 안의 문자 중 하나 선택 (예: [abc]는 a 또는 b 또는 c). 
        - {} : 콤마로 구분된 항목 중 하나 선택 (예: {jpg,txt}는 jpg 또는 txt).

- 사용 예시 
    - ls a? : a로 시작하는 두 글자 파일 검색. 
    - ls *[cd] : 파일명이 c 또는 d로 끝나는 모든 파일. 
    - touch {A..E}.txt : A.txt부터 E.txt까지 한 번에 생성.

문자열 검색 (grep)

- 기본 옵션 
    - -i : 대소문자 무시 (Ignore case). 
    - -n : 행 번호 출력 (Line number). 
    - -v : 패턴이 일치하지 않는 행만 출력 (Invert match). 
    - -w : 단어 단위로 정확히 일치할 때만 출력 (Word). 
    - -l : 문자열이 포함된 파일의 이름만 출력.

- 정규 표현식 (Regex) 
    - . : 임의의 한 문자. 
    - ^ : 줄의 시작 (예: ^root는 root로 시작하는 줄). 
    - $ : 줄의 끝 (예: bash$는 bash로 끝나는 줄). 
    - * : 앞 문자가 0번 이상 반복. 
    - + : 앞 문자가 1번 이상 반복 (egrep 전용).

파일 찾기 (find)

- 검색 조건 
    - -name : 파일 이름으로 검색. 
    - -type : 파일 종류 (f: 일반파일, d: 디렉터리). 
    - -size : 크기 검색 (+는 이상, -는 이하 / c: byte, k: KB). 
    - -mtime : 수정 시간 (일 단위).

- 동작 실행 (-exec) 
    - 구조: find [경로] [조건] -exec [명령어] {} \; 
    - {} : find로 찾은 결과값들이 들어가는 위치. 
    - \; : exec 옵션의 끝을 의미. 
    - -ok : 실행 전 사용자에게 매번 허가를 물어봄 (Interactive).

- VI 에디터 (Vim)

- 모드 전환
     - 명령 모드 : 초기 상태. ESC를 누르면 이 모드로 복귀. 
     - 입력 모드 : i 또는 a를 눌러 진입. 텍스트 작성 가능. 
     - 마지막 행 모드 : :를 눌러 진입. 저장 및 종료 실행.

- 저장 및 종료 
    - :w : 저장. 
    - :q : 종료. 
    - :wq : 저장 후 종료. 
    - :q! : 저장하지 않고 강제 종료.

## 🛠️ 3. 트러블슈팅 (Troubleshooting)
### 🔍 문제 발생 (Issue)
- **상황: 와일드카드를 포함한 find 명령어 실행 시 오류 발생** 
- **에러 메시지:** `ind: paths must precede expression: core2 / find: possible unquoted pattern after predicate -name?`

### 💡 원인 분석 (Cause)
- 원인 분석 (Cause)
    - find . -name core*와 같이 따옴표 없이 와일드카드를 사용함.
    - 쉘(Shell)이 find 명령어를 실행하기 전 현재 디렉터리에 있는 core1, core2 파일을 찾아 - - 명령어를 find . -name core1 core2 형태로 확장해버림.

find 명령어는 -name 인자로 하나의 패턴만 받을 수 있어 문법 오류가 발생함.

### 🚀 해결 방법 (Solution)
- 와일드카드 문자(*)가 쉘에 의해 미리 해석되지 않도록 작은따옴표(' ')로 감싸서 전달함
- 해결 명령어: find . -name 'core*' -ok rm {} \;
