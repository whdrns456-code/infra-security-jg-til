**핵심 개념 2: 퍼미션** 
- 파일 권한 및 소유권 관리
- chmod (Change Mode)   
    - 8진수 모드를 사용하여 파일/디렉터리의 권한을 변경함.
    - 형식: chmod [8진수] [파일/디렉터리]
    - chown (Change Owner)

- 파일이나 디렉터리의 소유자와 그룹을 변경함.
    - 소유자 변경: chown [user] [file]
    - 그룹만 변경: chown :[group] [file]
    - 소유자와 그룹 동시 변경: chown [user]:[group] [file]
    - 하위 디렉터리까지 일괄 적용: chown -R [user] [directory]

- chgrp (Change Group)
    - 그룹 소유권만 변경할 때 사용: chgrp [group] [file]

- umask와 기본 권한
    - 기본 개념
    - 파일이나 디렉터리 생성 시 부여되는 기본 권한을 결정함.
    - 시스템 기본값: 디렉터리는 777, 파일은 666에서 시작함.
    - umask 계산 및 설정
    - 계산법: 기본 권한에서 umask 비트가 켜진 부분의 권한을 제거함 (단순 뺄셈이 아닌 비트 마스킹).

- 명령어: umask (현재 값 확인), umask 027 (임시 변경).
- 특징: 터미널에서 직접 설정하면 재접속 시 초기화됨 (임시값).
- 영구 설정: /etc/profile 혹은 사용자의 .bashrc, .bash_profile 파일 수정 필요.

- 특수 권한 (Sticky Bit)
- /tmp 디렉터리 등에서 보이는 drwxrwxrwt의 t는 스티키 비트를 의미함.
- 누구나 쓸 수 있지만, 본인이 만든 파일만 지울 수 있게 제한하는 역할.
- **구글링해서 찾아본거라서 확신이 안든다.**

- 쉘 메타 문자 (Shell Meta Characters)
    - 디렉터리 관련
    - ~: 현재 로그인된 사용자의 홈 디렉터리.
    - ~[계정명]: 특정 사용자의 홈 디렉터리.
    - ~+: 현재 작업 디렉터리 (pwd와 동일).
    - ~-: 이전 작업 디렉터리.
    - cd -: 바로 직전에 있었던 디렉터리로 이동.

- 파일 대체 (와일드카드)
    - *: 0개 이상의 문자 대체.
    - ?: 딱 한 글자 대체.
    - []: 괄호 안의 문자 중 하나와 일치 (예: [abc]).
    - {}: 임의의 문자열 집합 생성 (예: touch {a,b,c}.txt).

- 인용 부호 (Quoting)
    - "": 대부분의 특수문자를 일반 문자로 취급하지만 $, `, \, !는 예외적으로 기능이 유지됨.

    - '': 모든 특수문자를 일반 문자로 취급.

    - \: 바로 뒤의 한 글자만 일반 문자로 취급.
    - ` ` (백틱): 명령어의 실행 결과를 출력함. $(명령어)와 동일한 기능.

- **핵심 개념 3: 입출력 재지정 (Redirection)** 
- 파일 디스크립터 (File Descriptor)
    - 프로세스가 파일에 접근하기 위해 부여하는 번호
    - 0 : 표준 입력 (Standard Input, 키보드)
    - 1 : 표준 출력 (Standard Output, 모니터)
    - 2 : 표준 에러 (Standard Error, 모니터)

- 재지정 활용 사례
- '>' (덮어쓰기) : 명령어 결과를 파일로 저장할 때 사용. 기존 내용은 삭제됨.
    - 예: ls -l > list.txt (파일 목록을 저장)
- '>>'(이어쓰기) : 기존 내용 뒤에 결과를 추가할 때 사용.
- 예: echo "new data" >> list.txt

- 2> (에러 재지정) : 정상 출력은 모니터에 보내고, 에러만 따로 파일에 담을 때 사용.

- 2> /dev/null : 에러 메시지를 화면에서 지우고 싶을 때 사용 (블랙홀로 폐기).
- 예: find / -name "passwd" 2> /dev/null
- < (입력 재지정) : 키보드 입력 대신 파일의 내용을 명령어의 입력으로 전달.
- 예: wc < /etc/hosts

- 고급 재지정
- 2>&1 : 에러(2)를 표준 출력(1)이 가는 곳으로 똑같이 보냄 (결과와 에러를 한 파일에 합치기).
- &> / &>> : 표준 출력과 에러를 한 번에 재지정하는 줄임표 표현.

### 💡 원인 분석 (Cause) 
- find / -name passwd > $(date+%Y%m%d).txt 시 나올 수 있는 에러
- 오늘 배운 입출력 재지정이다 오류 내용을 저장하거나 라는 강사님 얘기를 들었을 때

- 사실 배운 내용이 아니라서 제법 여러 번 오류가 발생했다 $()도 생략도 해보고 YYmmdd도 해보고 
- 2026Y02m05d.txt도 만들어보고 제법 아주 많이 골치가 아팠다 
### 🚀 해결 방법 (Solution)
- $(date +%Y%m%d).txt
- 해결 방법은 공백과 $()이다 
- 명령어 치환(Command Substitution)이라고 한다 ``과 같은 역할을 한다고 한다.
