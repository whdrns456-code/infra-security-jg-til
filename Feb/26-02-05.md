- **핵심 개념 1:** 
- vi 편집기 다루기
- 입력 모드 전환 (입력 시 필수)
    - i : 현재 커서 위치에서 편집 시작
    - I : 현재 행의 맨 앞으로 이동 후 시작
    - a : 현재 커서 한 칸 뒤에서 시작
    - o : 현재 행 아래에 새 줄 만들고 시작
    - O : 현재 행 위에 새 줄 만들고 시작

- 이동 및 편집 (명령 모드)

     - h, j, k, l : 왼쪽, 아래, 위, 오른쪽 이동

    - G : 문서 맨 아래로 이동
        - [숫자]G : 해당 행 번호로 이동 (예: 100G)
    - gg : 문서 맨 위로 이동
    - x : 현재 커서 글자 삭제 (Delete)
    - X : 커서 앞 글자 삭제 (Backspace)
    - dd : 현재 행 잘라내기 및 복사
    - yy : 현재 행 복사하기
    - p : 커서 아래에 붙여넣기
    - P : 커서 위에 붙여넣기
    - D : 커서 위치부터 해당 행 끝까지 삭제
    - u : 직전 작업 되돌리기 (Undo)
    - U : 해당 행에서 한 모든 작업 취소

- 검색 및 환경 설정
    - /문자열 : 아래 방향으로 문자열 검색 (n은 다음, N은 이전)
    - ?문자열 : 위 방향으로 문자열 검색
    - :wq : 저장 후 종료
    - :q! : 저장하지 않고 강제 종료
    - ZZ : 저장 후 즉시 종료 (대문자)
    - :!명령어 : vi 실행 중 외부 명령어 한 번 실행
    - :sh : 터미널 쉘로 잠시 이동 (exit 입력 시 vi 복귀)
    - :set nu : 줄 번호 표시
    - :set nonu : 줄 번호 숨기기

- 2. 파일 접근 및 실행 권한 제어
    - 퍼미션 구조 이해 (ls -l 확인)
        - [파일타입][소유자][그룹][기타사용자] 구조

- 파일타입 : - (일반 파일), d (디렉토리)
    - r (읽기) : 파일 내용 조회, 디렉토리 목록 확인(ls)
    - w (쓰기) : 파일 내용 수정, 디렉토리 내 파일 생성/삭제
    - x (실행) : 파일 실행(./파일명), 디렉토리 접근(cd)

- chmod 권한 변경 (심볼릭 모드)
    - u(소유자), g(그룹), o(기타), a(전체) 조합
    - chmod u+x hosts : 소유자에게 실행 권한 추가
    - chmod u-x hosts : 소유자 실행 권한 제거
    - chmod a-rwx hosts : 모든 사용자의 모든 권한 제거
    - chmod a= hosts : 모든 권한 초기화 (권한 없음 상태)
    - chmod 권한 변경 (숫자 모드)
    - r=4, w=2, x=1 합산 방식
    - chmod 755 hosts : 소유자(rwx), 그룹(rx), 기타(rx)
    - chmod 644 hosts : 소유자(rw-), 그룹(r--), 기타(r--)

- 권한 및 보안 핵심 사항
    - 보안 정의 : 허가받은 사람에게만 허가된 행위를 허용하는 것
    - 디렉토리 권한 : 디렉토리는 내부 진입 및 사용을 위해 r과 x 권한이 필수적임
    - 실행 방법 : 실행 권한이 있는 파일은 경로를 포함하여 실행 (예: ./hosts)

- Root 권한 : 소유자 계정인 root는 파일 권한 설정과 관계없이 모든 접근이 가능함
