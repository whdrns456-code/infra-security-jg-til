🔐 1. sudo 권한 (관리자 대장 되기)
기본 조건: wheel 그룹에 넣거나, /etc/sudoers.d/ 밑에 내 이름으로 파일 만들어서 설정하면 됨.

설정 한 줄 요약:

계정명 ALL=(ALL) ALL : 실행할 때 내 비번 물어봄.

계정명 ALL=(ALL) NOPASSWD: ALL : 비번 안 물어보고 프리패스.

기억할 것: root가 일반 사용자로 바꿀 땐 비번 따위 필요 없음. sudo -i는 아예 루트 쉘로 들어가는 거.

📁 2. 사용자 계정 & 설정
기본 설정: /etc/default/useradd랑 /etc/login.defs 파일에 계정 생성 규칙이 다 들어있음.

수정 팁: 코멘트 지우려면 usermod -c '' 하면 되고, 홈 디렉터리 꼬이면 chmod 700이랑 chown으로 소유권 다시 잡아주면 끝.

🛠️ 3. 리눅스 특수 권한 (s랑 t)
SetUID (4000): 파일 소유주(root) 권한으로 실행됨. 소유자 자리에 s 표시. passwd 명령어가 대표적.

SetGID (2000): 그룹 권한을 빌려 쓰거나, **디렉터리에서는 '상속'**이 핵심. 누가 파일을 만들어도 부모 폴더 그룹을 그대로 물려받음. 협업 폴더 만들 때 필수.

Sticky Bit (1000): 디렉터리에만 의미 있음. 누구나 만들 순 있지만, 삭제는 지 거만 가능. /tmp 같은 공용 폴더용.

대문자 S / T: 실행 권한(x)도 없으면서 특수 권한만 주면 뜨는 경고. "이거 작동 안 한다"는 뜻임.

🚦 4. 권한의 진짜 핵심
디렉터리 권한: r, w가 아무리 빵빵해도 x(실행) 권한 없으면 그 폴더에 들어가지도 못하고 꽝임. 무조건 x가 있어야 읽고 쓰고 다 할 수 있음.

8진수 계산: 특수 권한은 네 자리 숫자로! (SetUID=4, SetGID=2, Sticky Bit=1). 평소 쓰는 세 자리 앞에 하나 더 붙이면 됨.

1. 단일성 예약: at
특정 시점에 딱 한 번만 실행하고 싶을 때 사용한다. atd 서비스가 이 작업을 담당한다.

💡 주요 특징
터미널 독립성: 제어 터미널이 없어서 화면 출력을 할 수 없다. 결과는 메일로 받거나 **리다이렉션(>)**을 통해 파일로 저장해야 한다.

사용법: at [시간설정] 입력 후 at> 프롬프트에서 명령어를 치고 Ctrl + D로 저장한다.

📅 시간 지정(timespec) 예시
at now + 2hour: 지금부터 2시간 뒤

at 16:00: 오후 4시 (오늘 지났으면 내일)

at teatime: 오후 4시 (16:00)

at midnight: 새벽 12시 (00:00)

at 16:00 2022-02-23: 특정 날짜와 시간 지정

🛠 관리 명령어
atq (또는 at -l): 예약된 작업 목록 확인

at -c [번호]: 예약된 작업의 상세 내용 확인

atrm [번호]: 예약 취소

2. 반복성 예약: cron
주기적으로 반복되는 작업은 crond 서비스를 이용한다.

👤 사용자 cron (crontab)
사용자 개인의 반복 작업을 설정할 때 사용하며, 설정 시 5개의 시간 필드를 가진다.

[ 시간 설정 규칙 ]

Plaintext
 * * * * * [명령어]
분(0-59) 시(0-23) 일(1-31) 월(1-12) 요일(0-7)
*: 모든 값 (매분, 매시 등)

-: 범위 (15-18시)

/: 주기 (*/15 -> 15분마다)

,: 목록 (3,6,9 -> 3월, 6월, 9월)

[ 주요 명령어 ]

crontab -e: 작업 예약/수정 (vi 편집기 실행)

crontab -l: 예약 목록 확인

crontab -r: 모든 예약 삭제

crontab [파일명]: 작성해둔 파일의 내용을 예약으로 등록

🏢 시스템 cron & anacron
시스템 cron: /etc/crontab을 양식지로 사용하며, 사용자 계정 필드가 추가로 들어간다.

anacron: 시스템이 꺼져 있어 놓친 작업을 부팅 후에 다시 챙겨서 실행해 준다.

3. 실습 로그 (Review)
