1. RPM (Red Hat Package Manager)
RPM은 윈도우의 설치 파일(.exe)과 유사한 개념으로, 패키지 파일을 직접 다루는 도구다.

패키지 정보 확인 (Query 옵션)
설치된 패키지를 확인하거나 정보를 추출할 때 주로 사용한다.

rpm -qa: 시스템에 설치된 모든 패키지 목록 출력 (주로 grep과 함께 사용)

rpm -qi [패키지명]: 패키지의 상세 정보 확인

rpm -ql [패키지명]: 패키지에 포함된 모든 파일 리스트 출력

rpm -qc [패키지명]: 패키지의 설정(Config) 파일 리스트만 출력

rpm -qd [패키지명]: 패키지의 문서(Document) 파일 리스트 출력

rpm -qf [파일경로]: 특정 파일이나 디렉토리가 어떤 패키지에 의해 생성되었는지 확인

패키지 설치 및 삭제
설치/업데이트: rpm -Uvh [파일명.rpm]

U: 설치 또는 업데이트

v: 설치 과정 상세 확인

h: 설치 진행 상황을 # 마크로 표시

삭제: rpm -e [패키지명]

⚠️ RPM의 한계: 의존성(Dependency) 문제를 스스로 해결하지 못한다. A 패키지를 깔기 위해 B가 필요하다면, 사용자가 직접 B를 먼저 설치해야 한다.

2. YUM / DNF
인터넷 외부 저장소를 통해 패키지를 다운로드하고 설치하는 도구다. RPM의 의존성 문제를 자동으로 해결해 준다. (RHEL 8/Rocky 9부터는 DNF가 기본이지만 YUM과 혼용 가능하다.)

주요 명령어
install: 패키지 설치 (-y 옵션 사용 시 확인 절차 생략)

remove: 패키지 삭제

update: 패키지 업데이트

clean all: 기존의 캐시 데이터 삭제 (설정 변경 후 권장)

makecache: 현재 설정된 저장소의 메타데이터 캐시 생성

repolist all: 활성화/비활성화된 모든 저장소 리스트 확인

info: 패키지 상세 정보 확인

provides [파일경로]: 특정 파일을 포함하는 패키지 검색 (rpm -qf와 유사하지만 설치되지 않은 패키지도 검색 가능)

그룹 패키지 관리
특정 환경(예: 개발 도구, GUI 환경) 구축에 필요한 패키지 묶음을 관리한다.

yum groups list: 그룹 패키지 리스트 확인

yum groups install "[그룹명]": 그룹 패키지 전체 설치

3. 저장소(Repository) 이해 및 설정
Rocky Linux 9 등 최신 배포판은 시스템 안정성을 위해 저장소를 BaseOS와 AppStream으로 분리하여 관리한다.

저장소의 종류
BaseOS: OS 구동에 필수적인 커널 및 핵심 패키지. 하드웨어 지원 및 가상화 관련 필수 도구 포함.

AppStream: 사용자가 실제로 사용하는 애플리케이션(Vim, Apache), 개발 언어(Python, PHP), 데이터베이스(MariaDB) 등. 업데이트 주기가 BaseOS보다 빠르다.

.repo 파일 작성 요령
설정 파일은 반드시 /etc/yum.repos.d/ 디렉토리에 위치해야 하며 확장자는 .repo여야 한다.

- `오후 05시 36분 26초.`
- `일치하는 인수가 없습니다: telnet*`
- `오류: 일치하는 항목을 찾을 수 없습니다: telnet*`

### 💡 원인 분석 (Cause)
- 설정 파일 변경 시 시스템이 새로운 정보를 인식을 못 하는 문제가 발생하 
- 해당 패키지가 속한 저장소(보통 AppStream)가 enabled=0으로 비활성화

### 🚀 해결 방법 (Solution)
- 설정 파일을 변경하거나 생성했다면 yum makecache를 실행하여 시스템이 새 정보를 인식하게 해야 한다.
- 설치 실패 시 확인 사항: dnf install 시 패키지를 찾을 수 없다면, 해당 패키지가 속한 저장소(보통 AppStream)가 enabled=0으로 비활성화되어 있지 않은지 확인한다.
- 파일 위치 주의: .repo 파일은 반드시 /etc/yum.repos.d/ 아래에 있어야 시스템이 인식한다. 홈 디렉토리(~/) 등에서 작성한 경우 반드시 mv 명령어로 이동시켜야 한다
