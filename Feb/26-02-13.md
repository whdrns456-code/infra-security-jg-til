## 🛠️ 3. 트러블슈팅 (Troubleshooting)
- 🔍 문제 발생 (Issue)
    - 상황: yum makecache 실행 시 온라인 저장소(Net-BaseOS) 연결 실패

- 에러 메시지: `Curl error (60): SSL certificate problem: certificate is not yet valid`

- 💡 원인 분석 (Cause)
    - 시스템 시간 오류: 가상 머신(VM) 환경에서 재부팅 시 시스템 날짜가 과거(예: 2020년)로 초기화됨.

    - 인증서 거부: 웹 서버의 SSL 인증서는 2024~2026년에 유효한데, 클라이언트(내 PC) 시간이 그보다 이전이면 "아직 유효하지 않은 인증서"로 판단하여 보안상 연결을 차단함.

- 🚀 해결 방법 (Solution)
    - 수동 시간 업데이트: date 021310152026 명령어로 현재 시각을 강제 동기화함.

    - 영구 적용: hwclock --systohc를 통해 하드웨어 시계에 저장하거나 chronyd 서비스를 활성화하여 재부팅 시에도 인터넷 시간을 자동으로 받아오도록 설정함.
