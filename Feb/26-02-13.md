## 📝 1. 오늘의 학습 주제 및 핵심 내용
- **주요 주제: 네트워크 관리 기초 실습** 
- **핵심 개념 1:**
- 1. 내 컴퓨터 주소 확인 (조회)상태를 알아야 설정을 시작할 수 있다.
- ifconfig: 전통적인 방식. 보기에 깔끔하게 정리되어 나온다.
- ip addr: 현대 리눅스 표준. 아주 상세한 정보까지 다 보여준다.
- 특정 장치 확인: ip addr show ens224처럼 장치명을 붙여서 특정 인터페이스 정보만 쏙 뽑아볼 수 있다.
- 2. 네트워크 프로필 만들기 (Add)랜카드 하나에 여러 '설정 세트(프로필)'를 미리 만들어두고 필요할 때마다 갈아 끼울 수 있다.
    - ① 자동으로 주소 받기 (DHCP)공유기(DHCP 서버)가 주는 주소를 그대로 받아 쓰는 방식이다.Bashnmcli con add con-name dhcp type ethernet ifname ens224 con-name: 내가 부를 설정의 별명 (dhcp)ifname: 실제 장치 이름 (내 환경에서는 ens224)
    - ② 수동으로 주소 정하기 (Static/고정 IP)서버처럼 주소가 바뀌면 안 될 때 사용한다. 주소뿐만 아니라 나가는 문인 **게이트웨이(GW)**도 직접 알려줘야 한다.Bashnmcli con add con-name static2 type ethernet ifname ens224 ip4 192.168.56.200/24 gw4 192.168.56.1
- 💡 초보자 Tip: 192.168.56.200/24에서 /24는 내 동네 범위를 정하는 숫자다. (보통 255.255.255.0과 같은 의미)
- 3. 기존 설정 뜯어고치기 (Modify)이미 만들어진 설정을 개조하는 단계다. 오늘 실습에서 가장 난도가 높았던 부분이다.Bashnmcli con mod dhcp \
connection.id static \
ipv4.addresses 192.168.56.120/24 \
ipv4.gateway 192.168.56.1 \
+ipv4.addresses 192.168.56.30/24 \
ipv4.dns 8.8.8.8 \
connection.autoconnect no \
ipv4.method manual

- connection.id static: 별명을 dhcp에서 static으로 개명한다.
- +ipv4.addresses: 주소를 **추가(+)**한다. 
- 한 랜카드에 여러 IP를 쓸 수 있게 해준다.ipv4.dns 8.8.8.8: 구글 DNS를 설정한다. (인터넷 전화번호부 역할)
- ipv4.method manual: "이제 자동(DHCP) 안 쓰고 내가 직접 적은 수동 방식만 쓰겠다"는 선언이다.
- 4. 특정 주소 삭제 및 이름 변경설정을 빼거나 서버 이름을 바꾸는 마무리 작업이다.Bash# 특정 주소만 쏙 빼기 (-) nmcli con mod static -ipv4.addresses 192.168.56.120/24

- # 설정 적용하기 (반드시 UP을 해줘야 반영된다)
# nmcli con up static

- # 컴퓨터 이름(Hostname) 바꾸기
hostnamectl set-hostname nobreak.com

## 🛠️ 3. 트러블슈팅 (Troubleshooting)
- 🔍 문제 발생 (Issue)
    - 상황: yum makecache 실행 시 온라인 저장소(Net-BaseOS) 연결 실패

- 에러 메시지: `Curl error (60): SSL certificate problem: certificate is not yet valid`

- 💡 원인 분석 (Cause)
    - 시스템 시간 오류: 가상 머신(VM) 환경에서 재부팅 시 시스템 날짜가 과거(예: 2020년)로 초기화됨.

    - 인증서 거부: 웹 서버의 SSL 인증서는 2024~2026년에 유효한데, 클라이언트(내 PC) 시간이 그보다 이전이면 "아직 유효하지 않은 인증서"로 판단하여 보안상 연결을 차단함.

- 🚀 해결 방법 (Solution)
    - 수동 시간 업데이트: date 021310152026 명령어로 현재 시각을 강제 동기화함.

    - 영구 적용: hwclock --systohc를 통해 하드웨어 시계에 저장하거나 chronyd 서비스를 활성화하여 재부팅 시에도 인터넷 시간을 자동으로 받아오도록 설정함.
