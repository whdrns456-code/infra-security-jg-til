🖥️ ServerA 설정 요약 (명령어 및 순서)
1단계: 유동 IP 연결 생성 (dhcp1)
가장 먼저 ens224 장치에 대해 자동으로 IP를 할당받는 프로필을 생성했습니다.

명령어:

Bash
nmcli con add con-name dhcp1 type ethernet ifname ens224 ipv4.method auto
의미: ens224라는 랜카드에 dhcp1이라는 이름의 설정을 추가하고, IP는 자동으로 받겠다(auto)는 뜻.

2단계: 고정 IP 연결 생성 (static1)
지시서의 첫 번째 고정 IP(200번) 설정을 추가하고 활성화.

명령어:

Bash
nmcli con add con-name static1 type ethernet ifname ens224 \
ipv4.addresses 192.168.100.200/24 \
ipv4.gateway 192.168.100.2 \
ipv4.method manual

nmcli con up static1
의미: 200번 IP와 게이트웨이를 수동(manual)으로 설정한 프로필을 만들고, up 명령어로 즉시 적용.

3단계: 연결 설정 수정 (dhcp1 → static100)
이미 만들어둔 dhcp1 프로필의 이름과 속성(IP 150번, DNS 추가)을 변경

명령어:

Bash
nmcli con mod dhcp1 connection.id static100 \
ipv4.addresses 192.168.100.150/24 \
ipv4.gateway 192.168.100.2 \
ipv4.dns 8.8.8.8 \
ipv4.method manual

nmcli con up static100
의미: 기존 dhcp1의 이름을 static100으로 바꾸고, IP를 150으로 고정하며 DNS(8.8.8.8)를 추가한 뒤 적용했습니다. 이 과정에서 IP가 바뀌어 SSH 연결이 한 번 끊겼다

4단계: 호스트네임 설정
서버의 이름을 식별하기 좋게 변경

명령어:

Bash
hostnamectl set-hostname servera.est.org
의미: 시스템의 호스트네임을 servera.est.org로 영구적으로 변경합니다.

## 1. 개요 및 목적
본 실습은 가상화 환경(VMware) 및 클라우드(AWS/Oracle)를 활용하여 웹 서비스 운영을 위한 **3-Tier(Web-WAS-DB) 아키텍처**를 설계하고, 시스템 보안을 위한 **SSH 키 기반 인증** 및 **Firewalld 방화벽** 설정을 구현하는 것을 목적으로 한다.

## 2. 시스템 아키텍처 설계
서버 간 역할 분담을 통해 부하 분산과 보안성을 높인 3중 구조를 설계하였다.

| 서버 구분 | 호스트명 | IP 주소 | 주요 스택 | 역할 |
| :--- | :--- | :--- | :--- | :--- |
| **Web Server** | Server-Web | 192.168.100.150 | Nginx | 리버스 프록시, 로드밸런싱 |
| **App Server** | Server-App | 192.168.100.151 | Node.js / Java | 비즈니스 로직 처리 (WAS) |
| **DB Server** | Server-DB | 192.168.100.200 | MySQL / MariaDB | 데이터 저장 및 관리 |

## 3. 핵심 기술 구현 내용

### 🔒 3.1 SSH 접속 및 키 기반 인증
보안 강화를 위해 패스워드 인증 방식을 지양하고, 비대칭키 암호화 방식을 적용하였다.

**[실행 명령어]**
```bash
# 1. 클라이언트에서 키 쌍(공개키+개인키) 생성
ssh-keygen -t rsa -b 4096

# 2. 생성된 공개키를 서버에 등록 (서버의 패스워드 필요)
ssh-copy-id -i ~/.ssh/id_rsa.pub user@192.168.100.150

# 3. 이후 패스워드 없이 접속 확인
ssh user@192.168.100.150
