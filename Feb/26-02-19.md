🖥️ ServerA 설정 요약 (명령어 및 순서)
1단계: 유동 IP 연결 생성 (dhcp1)
가장 먼저 ens224 장치에 대해 자동으로 IP를 할당받는 프로필을 생성했습니다.

명령어:

Bash
nmcli con add con-name dhcp1 type ethernet ifname ens224 ipv4.method auto
의미: ens224라는 랜카드에 dhcp1이라는 이름의 설정을 추가하고, IP는 자동으로 받겠다(auto)는 뜻입니다.

2단계: 고정 IP 연결 생성 (static1)
지시서의 첫 번째 고정 IP(200번) 설정을 추가하고 활성화했습니다.

명령어:

Bash
nmcli con add con-name static1 type ethernet ifname ens224 \
ipv4.addresses 192.168.100.200/24 \
ipv4.gateway 192.168.100.2 \
ipv4.method manual

nmcli con up static1
의미: 200번 IP와 게이트웨이를 수동(manual)으로 설정한 프로필을 만들고, up 명령어로 즉시 적용했습니다.

3단계: 연결 설정 수정 (dhcp1 → static100)
이미 만들어둔 dhcp1 프로필의 이름과 속성(IP 150번, DNS 추가)을 변경했습니다.

명령어:

Bash
nmcli con mod dhcp1 connection.id static100 \
ipv4.addresses 192.168.100.150/24 \
ipv4.gateway 192.168.100.2 \
ipv4.dns 8.8.8.8 \
ipv4.method manual

nmcli con up static100
의미: 기존 dhcp1의 이름을 static100으로 바꾸고, IP를 150으로 고정하며 DNS(8.8.8.8)를 추가한 뒤 적용했습니다. 이 과정에서 IP가 바뀌어 SSH 연결이 한 번 끊겼을 것입니다.

4단계: 호스트네임 설정
서버의 이름을 식별하기 좋게 변경했습니다.

명령어:

Bash
hostnamectl set-hostname servera.est.org
의미: 시스템의 호스트네임을 servera.est.org로 영구적으로 변경합니다.
