- **주요 주제: DNS**
- **핵심 개념 1: DNS 개념 및 관련 파일**
    - 내용: IP Address를 Domain 형식(문자열) 변환해주는 서비스
    - /etc/hosts
        - 인터넷 주소와 호스트 이름의 관계를 설정
        - 해당 내용은 로컬에서만 유효하다. 

    - `/etc/resolv.conf` : 호스트가 사용할 DNS 서버의 주소를 설정 
        - 자신의 호스트가 DNS 서버라면 자신의 IP를 입력
        - nmcli로 gatway를 설정하면 자동으로 설정이 들어가는 경우가 발생할 수 있다. 
        - etc/resolv.conf를 vi나 ehco "192.168.100.150" > /etc/resolve.conf해당 파일을 수정을 해야된다. 
        - 
    - dns server 등록 예시 
    - nmcli device   [연결 가능한 device를 확인한다.]
    - nmcli con add con-name dns ifname ens192 ip4 192.168.100.150/24 ipv4.gateway 192.168.100.2  ipv4.dns 192.168.100.150  ipv4.method manual  
    - nmcli con up dns 

    - yum bind-utils -y install 
    - /etc/named.conf를 생성하기 위해서 먼저 bind를 install해야된다. 

    - /etc/named.conf 수정하기
    - listen-on port 53 { any; };
        - 외부 네트워크 대역에서 들어오는 모든 요청을 듣겠다
    - directory       "/var/named";       
    - allow-query     { any; };            # localhost 대신 any로 변경
        - 주소를 묻는 걸 허용한다
    - allow-transfer  { 192.168.100.200; }; # 슬레이브 서버 IP를 설정 
        - Zonfile 전체를 복사해 갈 수 있는 슬레이브 파일을 설정 

    - `zone "est.main" IN {
        type master;
        file "est.main.zone";};`
        - type : `master` 설정 
        - file : `/var/named/est.main.zone`에 주소 설정
    
    - systemctl restart named 설정 이후 나는 계속 까먹는다. 
    - systemctl 소프트웨어를 다시 시작하는 것 
    - nmcli con up dns (네트워크 관리) 

- /var/named/
    - named.empty  
    - cp -arp를 통해서 복사한다
    - -a (Archive) 옵션 : 파일의 속성 소유자 정보 생성 시간 등을 모두 그대로 유지 
- 일반적으로 복사를 시도하면 root권한 파일을 만들게 된다 named서비스는 이걸 읽지 못해 
    - "Permission Denied" 에러를 발생 시킨다. 
    --arp 복사를 하면 named.empty가 원래 가지고 있던 named 그룹 권한을 그대로 유지한 채 복사가 된다. 
    - `cp -arp named.empty est.main.zone`
