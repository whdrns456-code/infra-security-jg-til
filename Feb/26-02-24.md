## 📝 1. 오늘의 학습 주제 및 핵심 내용
- **주요 주제: Apache 가상 호스트(vHost) 설정**
- **핵심 개념 1:Virtual Host (가상 호스트)**
    - 하나의 웹 서버(하나의 IP)에서 여러 개의 도메인(vhost1, vhost2 등)을 운영하는 기술.
    - DocumentRoot와 ServerName 설정을 통해 클라이언트가 요청한 도메인에 맞는 디렉터리를 매핑함.

- `<VirtualHost *:80>`
    - 이 서버에 할당된 모든 IP(*)의 80번 포트(HTTP)로 들어오는 요청을 이 블록에서 처리하겠다. 

-  DocumentRoot /var/www/html1
    - 이 웹사이트의 실제 파일들이 저장된 폴더 경로입니다. 
    - 사용자가 접속했을 때 보여줄 index.html이 이 폴더 안에 있어야 된다.

-  ServerName vhost1.est.main
    - 이 가상 호스트가 응답할 대표 도메인 주소
    
- ServerAlias vhost1
대표 주소 외에 추가로 사용할 별칭입. vhost1이라고만 입력해도 같은 사이트로 접속.
`</VirtualHost>`

==> 서버에서 vhost1.est.main이라는 주소로 들어오면 var/www -/html1 폴더에 있는 파일을 보여줘라


- `<Directory /var/www/html1>` 섹션 
    - AllowOverride None
    - .htaccess 파일(개별 폴더 설정 파일)을 통한 설정을 허용하지 않겠다는 뜻 
    - 성능과 보안을 위해 보통 None으로 둔다.
   
- Require all granted
    - "모든 접속 요청을 허용하겠다" 이게 없으면 Apache는 보안을 위해 접속을 차단(403 Forbidden)한다.
- `</Directory>`

### 🛠️ 구현 결과
```language

# 1. 아파치 설치 및 방화벽 설정
yum -y install httpd
systemctl enable httpd --now
firewall-cmd --add-service=http --permanent && firewall-cmd --reload

# 2. 네트워크 다중 IP 설정 (ens160)
nmcli con add con-name web ifname ens160 type ethernet \
ip4 192.168.100.150/24 gw4 192.168.100.2 \
+ipv4.addresses 192.168.100.110/24 +ipv4.addresses 192.168.100.120/24
nmcli con up web

# 3. 가상 호스트 설정 (vhost1 예시)
# /etc/httpd/conf.d/01-vhost.conf 생성
<VirtualHost *:80>
    DocumentRoot /var/www/html1
    ServerName vhost1.est.main
    ServerAlias vhost1
</VirtualHost>


```

### ✨ 새롭게 알게 된 점
- 가상 호스트 강의를 들으면서 처음에는 이해가 안 갔다 
- 하지만 강의를 진행하고 구현한 명령어가 작동이 되었을 때 
- 매우 유용한 기술이라고 생각이 들었다 내가 생각한 방식은 가상호스트를 
이용해서 관리자 서버를 따로 운영하는 것이다. 
- 그렇게 하면 웹서버에 관리자 페이지를 두었을 때 발생할 수 있는 위협 그리고 코드들을 수정할 필요가 없어지게 된다 

-  백오피스(Back Office) 운용 시 장점 
    - 공격 표면(Attack Surface)의 최소화
    - 독립적인 보안 정책 적용
    - 시스템 부하 및 장애 격리 
    - 로그 분리

- 사실 단점은 아직 잘 모르갰다 
- 가상 호스트 개념을 듣고선 완벽할 거 같다는 생각에 너무 신나서 바로 개인 프로젝트 아키텍처도 만들었다.
