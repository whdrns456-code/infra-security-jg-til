# 📅 TIL: 2026-02-25

## 📝 1. 오늘의 학습 주제 및 핵심 내용
- **주요 주제: HAProxy를 이용한 SSL Termination 및 L7 로드밸런싱 구성**
- **핵심 개념 1: SSL Termination (SSL Offloading)**
    - 내용: 외부(Client)와 로드밸런서 사이는 HTTPS(443)로 암호화 통신을 하고, 로드밸런서와 내부 서버 사이는 HTTP(80) 평문으로 통신하여 서버의 부하를 줄이고 관리를 일원화하는 방식.
- **핵심 개념 2: Self-signed Certificate & PEM 파일**
    - 내용: 사설 인증서를 생성하고, HAProxy에서 사용하기 위해 인증서(.crt)와 개인키(.key)를 하나의 .pem 파일로 합쳐서 관리함.  
    - `/etc/pki/tls/private/`
- **핵심 개념 3: 포트 충돌 및 점유 확인**
    - 내용: 동일한 IP의 특정 포트(443)는 하나의 서비스만 점유할 수 있음. `ss -tulpn` 명령어를 통해 포트 점유 상태를 확인하고 충돌을 해결함.

## 💻 2. 실습 및 구현 내용
### ✅ 실습 목표
- HAProxy를 통해 1개 IP에서 80/443 포트를 리슨하고, 3개의 가상 IP로 운영되는 Apache 서버로 트래픽을 분산함.

### 🛠️ 구현 결과 (핵심 명령어 및 설정)


```bash
# 1. 인증서 생성 및 결합
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
-keyout /etc/ssl/server.key \
-out /etc/ssl/server.crt \
-subj "/C=KR/ST=Seoul/L=Seoul/O=JG/OU=IT/CN=192.168.100.136"

cat /etc/ssl/server.crt /etc/ssl/server.key > /etc/ssl/server.pem

# 2. HAProxy 설정 (/etc/haproxy/haproxy.cfg)
# frontend main
#     bind 192.168.100.136:80
#     bind 192.168.100.136:443 ssl crt /etc/ssl/server.pem
#     http-request redirect scheme https if !{ ssl_fc }
#     default_backend app
#
# backend app
#     balance roundrobin
#     server app1 192.168.100.230:80 check
#     server app2 192.168.100.240:80 check
#     server app3 192.168.100.250:80 check
✨ 새롭게 알게 된 점
SSL Termination의 효율성: 각 웹 서버마다 SSL 설정을 할 필요 없이 로드밸런서 한 곳에서만 관리하면 되므로 운영 효율이 극대화됨을 배움.

아파치 설정의 유연성: 확장자를 .bak으로 변경하여 불필요한 설정 파일(ssl.conf)을 손쉽게 비활성화하는 트릭을 익힘.

🛠️ 3. 트러블슈팅 (Troubleshooting)
🔍 문제 발생 (Issue)
상황: HAProxy 설정 후 재시작 시 Job for haproxy.service failed 에러 발생.

에러 메시지: cannot bind socket (Address already in use) for [192.168.100.136:443]

💡 원인 분석 (Cause)
기존에 설치된 mod_ssl 패키지로 인해 Apache(httpd)가 모든 인터페이스의 443 포트를 미리 점유하고 있었음.

🚀 해결 방법 (Solution)
grep -r "Listen 443" /etc/httpd/로 설정 위치 확인.

/etc/httpd/conf.d/ssl.conf의 확장자를 .bak으로 변경하여 Apache의 443 점유를 해제함.

systemctl restart httpd 후 HAProxy 재시작 성공.
