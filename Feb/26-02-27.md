## 📝 1. 오늘의 학습 주제 및 핵심 내용
- **주요 주제: NFS(Network File System)**
- **핵심 개념 1: nfs 개요**
    - 내용: 윈도우의 공유 폴도($)처럼 파일을 공유하여 사용할 수 있게 해준다. 
    - 로컬에 존재하는 파일을 공유할 수 있으며, 파일을 공유할 때 접근이 가능한 호스트 목록을 지정할 수 있다
    - 커버로스와 함께 사용하여 인증, 무결성 검증, 암호화 통신을 사용할 수 있다.
- **핵심 개념 마운트 방식 : **
    - 종류: 
        - 수동 마운트 (mount)	일회성, 테스트용	
        - 정적 마운트 (/etc/fstab)	부팅 시 자동 연결	
        - 자동 마운트 (autofs)	필요할 때만 연결	
- autofs 방식 : 
    - autofs는 사용자가 해당 디렉토리에 접근할 때만 마운트하고, 일정 시간 사용하지 않으면
    - 자동으로 umount되어서 자원을 아낄 수 있다
    - 와일드 카드를 사용하여 관리의 유연성을 더 할 수 있다

- 🛠️ 3. 트러블슈팅 (Troubleshooting)
- 🔍 문제 발생 (Issue)
상황: 클라이언트에서 마운트 시도 시 응답 없음 (Hang) 또는 권한 거부 발생.

에러 메시지: mount.nfs: Connection timed out 또는 Permission denied

- 💡 원인 분석 (Cause)
방화벽: 서버측에서 nfs, rpc-bind, mountd 서비스가 방화벽에 막혀 있음.

권한 미비: /etc/exports 설정 후 exportfs -r을 실행하지 않았거나, 실제 디렉토리의 시스템 권한이 없음.

- 🚀 해결 방법 (Solution)
    - showmount -e [서버IP] 명령어로 클라이언트에서 서버의 공유 목록이 보이는지 먼저 확인.
    - 서버에서 firewall-cmd --add-service={nfs,nfs3,mountd,rpc-bind} --permanent 실행 후 리로드.
