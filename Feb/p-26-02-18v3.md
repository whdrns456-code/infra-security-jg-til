### 🔍 문제 발생 (Issue)
- **상황:** 
- **에러 메시지:** `by .dockerignore: stat care-system-0.0.1-SNAPSHOT.jar: file does not exist`

-`address already in use`

### 💡 원인 분석 (Cause)
- 1. Dockerfile에서는 care-system-0.0.1-SNAPSHOT.jar라는 파일을 가져다 쓰겠다고 적혀 있는데, 지금 서버에는 이 파일이 생성되어 있지 않아서 도커가 "어라? 파일이 어디 있지?" 하고 멈춰버린 건다

- 2. 기존에 사용 중인 포트번호를 지워야 한다. 

- 3. Nginx (정지 및 비활성화)
- # 1. Nginx 서비스 정지
    - sudo systemctl stop nginx

- # 2. 부팅 시 자동 실행 방지 (선택 사항)
    - sudo systemctl disable nginx

- # 3. 80번 포트가 비었는지 다시 확인 (아무것도 안 나오면 성공!)
    - sudo lsof -i :80


- 이 때 진짜 미치고 싶었다 

### 🚀 해결 방법 (Solution)
- # 실행 권한 부여
    - chmod +x gradlew

- # 빌드 실행 (Clean 후 Build)
    - ./gradlew clean build -x test
    - 스프링 자체에서 test를 진행하지 않고 빌드를 진행하는거다 
    - 스프링 자체적으로 테스트를 진행하다가 문제가 발견하면 빌드가 되지 않아 해당 방법으로 진행을 했다. 
    - cp build/libs/care-system-0.0.1-SNAPSHOT.jar . 혹시 모를 내 위치로 jar 파일을 가지고 왔다. 

- 2 
    - sudo lsof -i :[포트번호] 을 통해서 해당 프로세스를 찾는다. 
    - sudo kill -9 [확인된_PID_번호] 해당 프로세스를 지운다 
    - 이미 사용 중인 프로세스를 지운다 내가 할당할 프로세스 세우기 위해서 
    - sudo docker-compose up -d
- # 컨테이너 상태 확인 (Up 8080->8080 인지 확인)
    - sudo docker ps
- # 스프링 부트 로그 확인
    - sudo docker logs -f spring-app

- nano docker-compose.yml [포트매핑 수정]
- ports:
      - "HTTP:내부 컨테이너"  # 외부(80)포트로 들어오면 컨테이너 내부(8080)로 전달해라!
- # 1. 기존 컨테이너 및 네트워크 깔끔하게 삭제
    - sudo docker-compose down

- # 2. 변경된 포트 설정으로 다시 가동
    - sudo docker-compose up -d
