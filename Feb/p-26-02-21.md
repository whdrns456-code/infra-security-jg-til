## 📝 1. 오늘의 학습 주제 및 핵심 내용
- **주요 주제: 다중 컨테이너 환경 구성** 
- **핵심 개념 1: Nginx를 활용한 로드밸런싱 및 고가용성(HA) 아키텍처 구현**
- 컨테이너 기반 인프라 구축: 
    - Docker를 활용하여 Nginx(Proxy), Spring Boot(App) 2중화, Oracle(DB) 등 총 5개 서비스를 컨테이너화하여 독립적인 운영 환경을 구축했습니다.

- L7 로드밸런싱 적용: 
    - Nginx를 단순 `Reverse Proxy`로 사용하는 것을 넘어, 
    - 백엔드 서버 그룹에 대한 **로드밸런싱(Load Balancing)**을 적용하여 트래픽 부하를 효율적으로 분산했습니다.

- 장애 극복(Failover) 및 고가용성 확보: 
    - 특정 WAS(Spring) 인스턴스에 장애가 발생하더라도 요청 우회 기능을 통해 서비스 중단 없이 운영되는 고가용성(High Availability) 환경을 검증했습니다.


- **핵심 개념 2: 시스템 아키텍처 다이어그램** 

[ 사용자 (Client) ]
       │
       ▼ (Port 80/443: HTTPS)
+------------------------------------------+
|          AWS EC2 (Host OS)               |
|  +------------------------------------+  |
|  |        Nginx (Load Balancer)       |  |
|  +---------+------------+-------------+  |
|            │            │                |
|      (backend-net: Docker Network)       |
|            │            │                |
|  +---------▼---+    +---▼---------+      |
|  |  Spring A   |    |  Spring B   |      |
|  | (App 8080)  |    | (App 8080)  |      |
|  +---------+---+    +---+---------+      |
|            │            │                |
|            ▼            ▼                |
|      +------------------------+          |
|      |    Oracle Database     |          |
|      +------------------------+          |
+------------------------------------------+

- **핵심 개념 3: 장점** 
     - 수평적 확장(Horizontal Scaling): 하드웨어 사양을 높이는 대신(Scale-up), 저렴한 서버 여러 대를 붙여서 전체 성능을 높일 수 있습니다.
     - 무중단 배포가 가능합니다 업데이트를 진행할 때마다 서버를 종료 시키지 않고 한 서버는 유지하면서 안정적인 업데이트가 가능합니다. 
     - 고가용성(High Availability)
