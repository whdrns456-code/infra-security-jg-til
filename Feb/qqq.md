1. 파일 한 번에 여러 개 만들기 (touch)
touch 명령어 뒤에 중괄호를 사용하여 규칙적인 이름을 가진 파일들을 순식간에 만들 수 있습니다.

연속된 숫자/알파벳 생성

touch file{1..10}.txt : file1.txt부터 file10.txt까지 10개 생성

touch {A..E}.log : A.log, B.log, C.log, D.log, E.log 생성

특정 이름 나열

touch apple.txt banana.txt cherry.txt : 공백으로 구분해서 나열

touch {test,sample,backup}.conf : test.conf, sample.conf, backup.conf 생성

조합해서 만들기 (오늘 실습 내용)

touch {A,B}.{jpg,pdf} : A.jpg, A.pdf, B.jpg, B.pdf 총 4개 생성


=============================================================================
여러 개 나열해서 만들기

mkdir dir1 dir2 dir3 : 현재 위치에 디렉터리 3개 생성

mkdir user{1..5} : user1부터 user5까지 디렉터리 5개 생성

중첩된 구조(하위 디렉터리) 한 번에 만들기 (-p 옵션)

mkdir -p music/2026/02

-p (parents) 옵션이 없으면 music 폴더가 없을 때 에러가 납니다.

-p를 쓰면 상위 폴더인 music과 2026까지 알아서 다 만든 뒤 최종적으로 02를 만듭니다.

복합 구조 만들기

mkdir -p project/{src,bin,lib,doc}

project라는 폴더를 만들고, 그 안에 4개의 하위 폴더를 동시에 생성합니다.

==================================
1. 완전히 다른 경로에 각각 만들 때 (공백 활용)
가장 단순한 방법은 mkdir 뒤에 원하는 경로들을 쭉 나열하는 것입니다.

명령어: mkdir /tmp/jpg /home/user/txt

/tmp 밑에는 jpg 폴더가 생기고, /home/user 밑에는 txt 폴더가 동시에 생깁니다.

2. 특정 부모 디렉터리 안에 여러 개를 만들 때 (중괄호 활용)
만약 /project라는 폴더 안에 jpg와 txt를 동시에 만들고 싶다면 중괄호를 쓰는 게 훨씬 빠릅니다.

명령어: mkdir -p /project/{jpg,txt}

결과: /project/jpg와 /project/txt가 생성됩니다.

-p 옵션을 붙여주면 만약 /project 폴더가 아직 없더라도 상위 폴더까지 알아서 다 만들어줍니다.

======================================================
명령어: mkdir -p a/b/c

설명:

만약 a 폴더가 없어도 a를 먼저 만들고,

그 안에 b를 만들고,

최종적으로 c까지 순차적으로 생성합니다.

이미 폴더가 존재하는 경우에도 에러를 내지 않고 그대로 넘어갑니다.

한 번에 이동 시키기


현재 위치에 있는 파일들을 각각 지정된 디렉터리로 한 번에 이동시키려면, **와일드카드(*)**를 사용하여 파일 확장자별로 mv 명령어를 실행하면 됩니다.

1. 한 줄씩 실행하는 방법 (가장 확실함)
가장 직관적인 방법은 각 확장자별로 명령어를 두 번 나누어 치는 것입니다.

txt 파일 이동: mv *.txt /project/txt/

jpg 파일 이동: mv *.jgp /project/jpg/

(참고: 아까 오타로 만드신 file10.jgp 등 .jgp 확장자에 맞춰서 입력하셔야 합니다!)

2. 한 줄로 묶어서 실행하는 방법
세미콜론(;)을 사용하면 두 명령어를 한꺼번에 내릴 수 있습니다.

명령어: mv *.txt /project/txt/ ; mv *.jgp /project/jpg/
