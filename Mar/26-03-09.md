## 📝 1. 핵심 이론 정리
- **OSI 7 Layer & PDU:** - L2(Data Link): Ethernet Header (Frame) - MAC 주소 기반 통신.
    - L3(Network): IP Header (Packet) - IP 주소 기반 통신.
    - L4(Transport): TCP/UDP Header (Segment) - Port 번호를 통한 서비스 연결.
- **Encapsulation:** 데이터(Payload)에 계층별 헤더 정보를 덧붙여 전송 가능한 형태로 만드는 과정.
- **통신 시나리오 (PC -> Naver):** 1. ARP 요청(게이트웨이 MAC 파악) -> 2. DNS 요청(Naver IP 파악) -> 3. TCP 3-Way Handshake(세션 연결) -> 4. HTTP Request/Response.

## 📝 1. 장비별 데이터 처리 방식
- **Hub (L1):** 전기 신호를 단순히 복제하여 모든 포트로 전송(Flooding). 보안성이 전혀 없음.
- **Switch (L2):** `MAC Address Table`을 참조하여 목적지로만 전달(Forwarding). 
    - **동작 원리:** Learning(학습) -> Flooding(모르면 뿌림) -> Forwarding(전달) -> Filtering(차단) -> Aging(삭제).
- **Router (L3):** `IP Routing Table`을 기반으로 서로 다른 네트워크(Subnet) 간 데이터 중계.

## 🧐 2. 보안적 성찰
- 스위치는 효율적이지만, **MAC Flooding Attack**을 받으면 주소 테이블이 포화되어 허브처럼 동작(Flooding)하게 되고, 이 과정에서 패킷 도청이 가능해진다는 취약점을 학습함.
## 📝 1. 주요 공격 유형
- **Sniffing:** 비암호화 프로토콜 트래픽을 몰래 훔쳐보는 행위. (대응: 암호화 프로토콜 사용)
- **Spoofing:** IP, ARP, DNS를 속여 데이터 흐름을 왜곡.
- **MITM (Man-In-The-Middle):** 중간자 공격을 통한 세션 하이재킹 및 데이터 조작.
- **DoS/DDoS:** 비정상적인 요청을 폭주시켜 서비스 마비 유도.

## 💻 1. 실습 내용

    - **Promiscuous Mode (무차별 모드):** 목적지에 상관없이 해당 세그먼트의 모든 트래픽을 수집. 도청 및 정밀 분석 시 필수적.
