📝 1. 오늘의 학습 주제 및 핵심 내용
주요 주제: 네트워크 가로채기(Sniffing/Spoofing) 및 TCP 세션 메커니즘 분석

핵심 개념 1: 스푸핑(Spoofing) 공격 기법

ARP Spoofing: IP주소에 대응하는 MAC 주소를 공격자의 주소로 속여 패킷을 가로채는 기법. arpspoof와 fragrouter를 활용해 중간자 공격(MITM) 구현.

DNS Spoofing: 희생자의 DNS 쿼리에 대해 위조된 응답을 먼저 보내 가짜 사이트(Phishing)로 유도.

IP Spoofing: 출발지 IP를 속여 공격자의 위치를 숨기거나 인증을 우회.

핵심 개념 2: TCP 세션 구조 분석

3-Way Handshake와 데이터 전송 과정에서 Sequence Number와 Acknowledgment Number의 동기화 원리 학습.

데이터 크기(Payload Size)에 따른 번호 변화 규칙 파악: Ack = Received Seq + Data Size + 1.

💻 2. 실습 및 구현 내용
✅ 실습 목표
Kali Linux를 공격자로 사용하여 Windows 10과 Rocky Linux 간의 통신 가로채기.

netcat(nc) 통신 패킷을 Wireshark로 캡처하여 TCP 번호 체계 정밀 분석.

🛠️ 구현 결과 (핵심 명령어)
Bash
# 1. ARP Spoofing (두 터미널에서 교차 실행)
sudo arpspoof -i eth0 -t 192.168.101.133 192.168.101.129
sudo arpspoof -i eth0 -t 192.168.101.129 192.168.101.133

# 2. IP Forwarding (패킷을 드롭하지 않고 전달)
sudo fragrouter -B1

# 3. DNS Spoofing
sudo dnsspoof -f /tmp/host

# 4. TCP 스캔 (hping3 활용)
sudo hping3 -p 80 -S [Target_IP]  # TCP SYN 스캔
✨ 새롭게 알게 된 점
Fragrouter의 중요성: ARP 스푸핑 시 공격자가 패킷을 가로채기만 하고 전달(forwarding)하지 않으면 희생자는 인터넷 연결이 끊겨 공격을 눈치채게 됨.

TCP Sequence Number: 단순히 순서만 정하는 것이 아니라 데이터의 바이트 단위 누적합을 통해 전송 무결성을 보장함을 패킷 분석을 통해 확인.

🛠️ 3. 트러블슈팅 (Troubleshooting)
🔍 문제 발생 (Issue)
상황: ARP 스푸핑 성공 후 희생자(Win10)에서 ping 응답이 오지 않음.

에러 메시지: Request timed out

💡 원인 분석 (Cause)
Kali Linux에서 IP Forwarding 설정이 활성화되지 않아 패킷이 공격자 PC에서 목적지로 전달되지 못함.
