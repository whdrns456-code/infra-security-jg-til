📝 1. 오늘의 학습 주제 및 핵심 내용
주요 주제: 리눅스 X 윈도(X Window) 시스템 및 데스크톱 환경

핵심 개념 1: X 윈도 구조 및 계보

클라이언트/서버 구조로 동작하며, 원격지의 응용 프로그램도 로컬 화면에 출력 가능함.

X 서버는 입력 장치(키보드, 마우스)와 출력 장치(모니터)를 관리하고, X 클라이언트는 실제 응용 프로그램을 의미함.

핵심 개념 2: 디스플레이 매니저와 윈도 매니저

디스플레이 매니저(GDM, SDDM): 사용자 로그인 및 세션 시작 관리.

윈도 매니저(Mutter, KWin): 창의 배치, 크기 조절, 테두리 등 외형 관리.

💻 2. 실습 및 구현 내용
✅ 실습 목표
부팅 타겟 변경 및 X 윈도 수동 실행 확인

🛠️ 구현 결과
Bash
# 기본 부팅 모드를 텍스트 모드로 변경
systemctl set-default multi-user.target

# 현재 부팅 모드 확인
systemctl get-default

# 텍스트 모드에서 X 윈도 수동 실행
startx
✨ 새롭게 알게 된 점
GNOME은 GTK+ 라이브러리를 사용하고 gedit, eog(이미지 뷰어)를 기본으로 함.

KDE는 Qt 라이브러리를 사용하며 Kate, Gwenview를 기본으로 함. (Gwenview는 G로 시작하지만 KDE 소속인 점 주의)

🛠️ 3. 트러블슈팅 (Troubleshooting)
🔍 문제 발생 (Issue)
상황: 텍스트 모드 부팅 후 systemctl xinit 명령어로 X 윈도 실행 시도 중 오류 발생.

에러 메시지: Unknown operation xinit

💡 원인 분석 (Cause)
systemctl은 서비스 유닛이나 타겟 제어용이지 개별 바이너리 실행용 명령어가 아님.

🚀 해결 방법 (Solution)
단독 스크립트 명령어인 startx를 사용하여 해결.
