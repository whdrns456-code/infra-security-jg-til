주요 주제: 리눅스 환경에서의 FTP 서버 구축 및 설정 (vsftpd)

핵심 개념 1: FTP (File Transfer Protocol)

내용: TCP/IP 프로토콜을 가지고 서버와 클라이언트 사이의 파일을 전송하기 위한 표준 프로토콜. 기본적으로 21번 포트를 제어용으로 사용함.

핵심 개념 2: vsftpd (Very Secure FTP Daemon)

내용: 보안성과 성능을 강조한 리눅스용 FTP 서버 소프트웨어. 설정이 간결하고 속도가 빨라 보안이 중요한 인프라 환경에서 주로 사용됨.

# 1. vsftpd 패키지 설치
sudo yum install -y vsftpd

# 2. 설정 파일 수정 (/etc/vsftpd/vsftpd.conf)
# anonymous_enable=NO  (익명 접속 비허용)
# local_enable=YES     (로컬 사용자 접속 허용)
# write_enable=YES     (쓰기 권한 허용)
# chroot_local_user=YES (사용자 홈 디렉토리 상위 이동 제한)

sudo vi /etc/vsftpd/vsftpd.conf

# 3. 서비스 시작 및 활성화
sudo systemctl start vsftpd
sudo systemctl enable vsftpd

# 4. 방화벽 설정 (FTP 서비스 허용)
sudo firewall-cmd --permanent --add-service=ftp
sudo firewall-cmd --reload

🛠️ 3. 트러블슈팅 (Troubleshooting)
🔍 문제 발생 (Issue)
상황: FTP 접속 시 '500 OOPS: vsftpd: refusing to run with writable root inside chroot()' 에러 발생.

에러 메시지: 500 OOPS: vsftpd: refusing to run with writable root inside chroot()

💡 원인 분석 (Cause)
chroot_local_user=YES 설정 시, 보안상의 이유로 사용자의 홈 디렉토리에 쓰기 권한이 있으면 접속을 거부하는 vsftpd의 기본 정책 때문임.

🚀 해결 방법 (Solution)
설정 파일에 allow_writeable_chroot=YES 옵션을 추가하거나, 홈 디렉토리의 쓰기 권한을 제거하여 해결함.
