## 📝 1. 오늘의 학습 주제 및 핵심 내용
- **주요 주제: 네트워크 해킹 기법 및 Windows 인증 시스템의 이해**
- **핵심 개념 1: Network Attack (Sniffing, Spoofing, MITM)**
    - **Sniffing:** 비암호화 프로토콜 도청. Hub 환경의 Flooding 특성을 이용하거나, Switch 환경에서 NIC의 Promiscuous mode를 활성화하여 가로챔.
    - **Spoofing:** IP/ARP 주소를 속여 데이터 흐름을 왜곡. 특히 ARP Poisoning은 Victim의 Cache Table을 조작하여 MITM 공격의 교두보 역할을 함.
    - **MITM (Man-In-The-Middle):** 중간자 공격. SSL Strip을 통해 HTTPS(443) 세션을 HTTP(80)로 강제 강하시켜 평문 데이터를 탈취.
- **핵심 개념 2: Windows 인증 아키텍처**
    - **LSA (Local Security Authority):** `lsass.exe` 프로세스로 동작하며 인증, 보안 정책, 액세스 토큰 생성을 총괄.
    - **SID (Security Identifier):** 사용자를 식별하는 고유 ID (예: Admin-500, 일반계정-1000번대).
    - **인증 프로토콜:** LM(취약) -> NTLM v1 -> NTLM v2(Challenge-Response 방식)로 진화.

## 💻 2. 실습 및 구현 내용
### ✅ 실습 목표
- Mimikatz 및 Pwdump를 이용한 자격 증명 추출 및 암호 크랙 메커니즘 이해

### 🛠️ 구현 결과
#### 1. Mimikatz (메모리 내 자격 증명 추출)
- `privilege::debug`: 디버그 권한 획득
- `sekurlsa::logonpasswords`: 메모리에 로드된 NTLM 해시 및 wdigest 평문 암호 확인

#### 2. Pwdump & Ophcrack (로컬 해시 크랙)
- `pwdump8.exe > result_win10.txt`: SAM 파일 등에서 계정별 해시값 추출
- **Ophcrack 연동:** 추출된 파일 끝에 `:::` 접미사를 추가하여 Kali Linux에서 로드 후 **Rainbow Table**로 암호 해독 수행

### ✨ 새롭게 알게 된 점
- **보안의 책임추적성:** 식별(ID) -> 인증(PW) -> 인가(Token) -> 감사(Log)로 이어지는 흐름을 실제 프로세스(`lsass.exe`, `sam`)와 매칭하여 이해함.
- **최신 DoS 동향:** KISA 가이드를 통해 HTTP GET Method Slowloris, RUDY 등 L7 영역의 공격 방식을 학습함.

## 🛠️ 3. 트러블슈팅 (Troubleshooting)
### 🔍 문제 발생 (Issue)
- **상황:** Pwdump로 추출한 해시 파일을 Kali의 Ophcrack에서 로드할 때 "파일 인식 실패" 발생

### 💡 원인 분석 (Cause)
- Pwdump8의 출력 포맷이 Ophcrack이 요구하는 표준 Pwdump 형식(마지막 구분자 포함)과 일치하지 않음.

### 🚀 해결 방법 (Solution)
- `result_win10.txt` 내 각 계정 정보 끝에 `:::`를 수동으로 추가하여 포맷팅을 수정 후 정상 로드 완료.

## 🧐 4. 오늘의 회고
### 💡 성찰 및 성장
- 인프라 보안의 핵심은 결국 '권한 관리'와 '암호화'임을 깨달음. 특히 Pass-the-Hash 공격처럼 비밀번호를 몰라도 해시값만으로 인증이 가능하다는 점이 충격적이었음.
- 15대 VM 프로젝트를 진행하며 구축했던 서버들의 보안 설정을 다시 점검해야겠다는 동기부여가 됨.
