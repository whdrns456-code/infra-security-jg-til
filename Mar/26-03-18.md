## 📝 1. 오늘의 학습 주제 및 핵심 내용
- **시스템 해킹의 기초: 메모리 구조와 실행 흐름 제어 취약점**
- **핵심 개념 1: 버퍼 오버플로우 (Buffer Overflow)**
    - 정의: 데이터 입력 시 설정된 버퍼(Buffer)의 경계를 넘어 인접한 메모리(RET, RBP 등)를 덮어쓰는 취약점.
    - 핵심: 함수의 반환 주소(Return Address)를 변조하여 프로그램의 흐름을 공격자의 의도대로 변경.
- **핵심 개념 2: 레이스 컨디션 (Race Condition) & TOCTOU**
    - 정의: 둘 이상의 프로세스가 공유 자원에 동시 접근 시 실행 순서에 따라 결과가 변하는 상태.
    - TOCTOU: 검사 시점(Time of Check)과 사용 시점(Time of Use) 사이의 간극을 이용한 권한 탈취.

🛠️ 3. 트러블슈팅 (Troubleshooting)
🔍 문제 발생 (Issue)
상황: 패키지 설치 중 apt lock 에러 발생.

에러 메시지: Waiting for cache lock: Could not get lock /var/lib/dpkg/lock-frontend

💡 원인 분석 (Cause)
백그라운드에서 자동 업데이트가 실행 중이거나 이전 설치 작업이 비정상 종료되어 잠금 파일이 남음.

🚀 해결 방법 (Solution)
ps aux | grep apt로 프로세스 확인.

프로세스가 없을 경우 sudo rm /var/lib/dpkg/lock-frontend 실행 후 sudo dpkg --configure -a로 패키지 매니저 복구.
