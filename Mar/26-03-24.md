    - **Error-based:** `HAVING`, `GROUP BY` 구문을 악용해 DB 에러 메시지로부터 테이블 및 컬럼 정보 획득.
    - **Union-based:** `UNION SELECT`를 사용하여 기존 쿼리 결과에 공격자가 원하는 타 테이블 데이터를 합쳐서 출력.

## 💻 2. 실습 및 구현 내용
### ✅ 실습 목표
- DVWA(Linux/PHP) 환경의 Brute Force 공격 및 Burp Suite 활용.
- Flower(Windows/ASP) 환경의 Error-based SQLi를 통한 회원 DB 탈취.

### 🛠️ 주요 공격 구문 (MS-SQL/Flower 기준)
1. **HAVING을 이용한 정보 수집**
   - `' HAVING 1=1--` 입력 시 에러를 통해 테이블명(`member`)과 첫 번째 필드(`m_idx`) 확인.
2. **GROUP BY를 이용한 컬럼 확장**
   - `' GROUP BY member.m_idx, member.m_id--` 와 같이 순차적으로 필드명을 찾아냄.
3. **UNION SELECT 데이터 탈취 (필드 수 7개 일치)**
   - `' union select NULL,0,NULL,m_id,m_pwd,NULL,NULL from member--`
   - 게시판 리스트의 제목/작성자 위치에 실제 회원 ID와 PW가 노출됨을 확인.

## 🛠️ 3. 트러블슈팅 (Troubleshooting)
### 🔍 문제 발생 (Issue)
- **상황:** `UNION SELECT` 시 데이터가 출력되지 않고 오류 메시지 발생.
- **에러 메시지:** `All queries combined using a UNION ... must have an equal number of expressions.`

### 💡 원인 분석 (Cause)
- 원본 게시판 쿼리가 불러오는 컬럼의 개수와 공격 쿼리의 컬럼 개수가 불일치함.

### 🚀 해결 방법 (Solution)
- `ORDER BY` 절로 컬럼 개수를 먼저 파악하거나, `NULL`의 개수를 조절하여 정상 응답이 오는 지점(7개)을 맞춰 해결.

## 🧐 4. 오늘의 회고
### 💡 성찰 및 성장
- 비전공자로서 인프라 보안을 배우며 DB 구조가 다소 생소했지만, 직접 에러를 유도하며 구조를 파악해 보니 SQL 쿼리의 논리적 흐름이 훨씬 명확해졌음.
- Linux(DVWA)와 Windows(Flower) 환경별로 주석처리(`#` vs `--`) 및 공격 패턴이 다름을 체득함.

- **개선 방안:** 주요 DB별 시스템 테이블 구조를 별도로 정리하여 암기할 것.
- **내일의 목표:** Blind SQL Injection의 개념과 시간 기반(Time-based) 공격 기법 학습.
