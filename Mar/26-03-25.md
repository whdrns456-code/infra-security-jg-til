## 📝 1. 오늘의 학습 주제 및 핵심 내용
- **주요 주제: Web Application Architecture 및 핵심 웹 취약점 (SQLi, XSS) 실습**
- **핵심 개념 1: HTTP 웹 애플리케이션 구조 및 프록시 통신**
    - 내용: HTTP Request/Response의 구조(Header, Body), GET/POST 메서드의 차이점을 복습함. 정상적인 TLS 통신(인증서 기반 암호화/복호화)의 흐름과 Burp Suite 같은 프록시 도구를 활용해 중간에서 세션을 가로채고 조작하는 원리를 이해함.
- **핵심 개념 2: SQL Injection (SQLi) 및 자동화 도구 활용**
    - 내용: Error Based, Blind(참/거짓 기반), Union 기반 등 다양한 SQLi 기법을 다룸. 특히 Blind SQLi에서 `ASCII(SUBSTR())` 함수를 이용해 데이터베이스 이름, 테이블, 컬럼명을 한 글자씩 유추하는 논리를 파악함. SQLMap을 활용해 공격을 자동화하고 해시 크랙까지 수행하는 과정을 실습함.
- **핵심 개념 3: XSS (Cross Site Scripting)**
    - 내용: 서버가 아닌 **사용자의 브라우저**를 타겟으로 악성 스크립트를 실행시키는 공격. 서버에 스크립트가 영구적으로 저장되는 Stored XSS와 URL 파라미터를 통해 즉각적으로 반사되는 Reflected XSS의 흐름을 비교하고, `document.cookie` 탈취 기법을 실습함.
