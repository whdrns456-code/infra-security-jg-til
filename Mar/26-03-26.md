# 📅 TIL: 2026-03-26 (인프라 보안 및 웹 해킹 실습)

## 📝 1. 오늘의 학습 주제 및 핵심 내용
- **주요 주제: 웹 애플리케이션 보안 취약점 분석 및 대응 방안**
- **핵심 개념 1: 클라이언트 측 공격 (XSS, CSRF)**
    - **DOM 기반 XSS:** 서버 응답과 관계없이 브라우저 단에서 JS가 DOM을 조작할 때 발생하는 취약점. innerHTML 대신 textContent 사용 권장.
    - **CSRF (Cross Site Request Forgery):** 사용자의 세션 권한을 도용하여 의도치 않은 요청(정보 수정, 송금 등)을 강제로 실행하게 함.
- **핵심 개념 2: 서버 측 공격 (Injection, File Inclusion, File Upload)**
    - **File Inclusion (LFI/RFI):** 서버 내부(Local) 또는 외부(Remote) 파일을 실행시켜 기밀 정보를 유출하거나 악성 코드를 실행.
    - **File Upload:** 웹쉘(WebShell) 업로드를 통한 서버 제어 권한 탈취 및 리버스 쉘(Reverse Shell) 연결.
- **핵심 개념 3: 인증 및 세션 보안**
    - **HttpOnly:** 자바스크립트를 통한 쿠키 탈취 차단.
    - **CSP (Content Security Policy):** 허용된 도메인에서만 스크립트 실행을 허용하는 브라우저 보안 정책.

🛠️ 3. 트러블슈팅 (Troubleshooting)
🔍 문제 발생 (Issue)
상황: DVWA에서 File Inclusion 실습 시 "allow_url_include NOT enabled" 경고 발생 및 기능 작동 안 함.

에러 메시지: Vulnerability: File Inclusion = off

💡 원인 분석 (Cause)
PHP 설정 파일(php.ini)에서 외부 URL을 include 하는 옵션이 보안상의 이유로 기본값 Off로 설정되어 있었음.

🚀 해결 방법 (Solution)
/etc/php.ini 파일 수정: allow_url_include = On 설정.

서비스 재시작: sudo systemctl restart httpd.service 및 php-fpm.service.
